Section 01

What the Machine Actually Reads.

When an AI processes a PDF it does not read the visual page. It reads the tag tree, a hidden structural layer embedded in the file that describes content in sequence. The visual layout is for humans. The tag tree is for machines.

Unstructured PDF

A two-column report with no tag structure might be read as:

"Q3 Revenue Our customers reported that satisfaction scores..." (jumps columns line by line)

Wrong reading order produces meaningless fragments. The AI either hallucinates to fill the gaps or returns an incomplete answer.

Well-Tagged PDF

With correct reading order and tags, the machine receives:

"Q3 Revenue. Total revenue for the quarter reached... [full column 1 complete] ...Our customers reported satisfaction scores of..." [full column 2]

Same PDF. Same visual layout. Completely different machine experience.

Section 02

How LLMs Ingest PDF Data.

The way an AI reads your PDF depends entirely on the type of tool. Most AI chat tools extract text in a single pass and reason over it in one session. Enterprise RAG systems go further, but they introduce a different challenge.

When a RAG system ingests a document, it breaks it into smaller pieces before indexing. If those breaks fall in the wrong place, a chunk might contain a data point with no heading to explain what it refers to. Retrieved out of context, that chunk produces a wrong or misleading answer.

One-Off Chat Upload

Claude, Copilot, ChatGPT
How it reads your PDF
Extracts text in one pass. Multi-column layouts, tables, and complex formatting often extract poorly.
Impact of poor structure
Garbled input means incomplete or out-of-sequence answers.

Enterprise RAG System

Acrobat AI Assistant, Copilot
How it reads your PDF
Ingests once, breaks into chunks, stores in a searchable index. Retrieves relevant chunks per question.
Impact of poor structure
This is where structure matters most. A chunk split mid-sentence or split from its heading loses all meaning when retrieved.
Section 03

Making PDFs AI-Ready: Step by Step.

1
Start in the Source Document
⌄
  • Use Heading 1, 2, 3 styles in the Styles panel, never bold or resize text to fake a heading
  • Keep layouts single-column where possible
  • Build tables with clear column headers and simple row structure
  • Add alt text to any image that carries meaning
Watch: Prepping Source Files for Accessibility
2
Run the Make Accessible Guided Action Acrobat Pro Required
⌄

In Acrobat Pro: All Tools > Use Guided Actions > Make Accessible. This automated action handles the foundational work before you do any manual review.

  • Adds document title and description
  • Runs OCR on scanned pages
  • Auto-tags the document structure
  • Sets the reading language
  • Prompts for alt text on images
  • Runs the Accessibility Checker at the end and flags what still needs attention
Why this matters for ingestion

Without tags: An untagged PDF is a flat string of text to any AI. There is no hierarchy, no section boundaries, no way to know where one topic ends and another begins. Chunks get formed arbitrarily and lose all meaning.

With tags: A tagged document gives the AI a map. Chunks can form at heading boundaries and paragraph breaks, which are meaningful units. Every retrieved piece of content still knows what section it belongs to.

Watch: Let Acrobat help you make accessible PDFs
3
Review and Fix Reading Order Acrobat Pro Required
⌄

In Acrobat Pro: All Tools > Prepare for Accessibility > Fix Reading Order. The tool overlays numbered boxes on the page. Those numbers are the exact sequence the machine reads your content in.

  • Check that numbers follow a logical path through the document
  • For multi-column layouts: confirm it reads all of column 1 before moving to column 2
  • Reorder boxes by dragging to correct any sequence errors
  • Tag headers, footers, and decorative elements as Background/Artifact to exclude them from the reading sequence
Why this matters for ingestion

Reading order is what determines how chunks form.

If the tag tree reads across two columns line by line, a chunk might start mid-sentence in column one and finish mid-sentence in column two. That chunk is meaningless when retrieved and will produce a confused or wrong answer.

Correct reading order means chunks form along the natural flow of the document, so retrieved content is always coherent and in context.

4
Verify and Fix Tags Acrobat Pro Required
⌄

Open the Tags panel: View > Show/Hide > Navigation Panes > Tags.

  • Confirm heading hierarchy: H1 for title, H2 for sections, H3 for subsections. No skipped levels.
  • Verify tables have TH (header) and TD (data) cells correctly assigned. Right-click a tag > Table Cell Properties to fix.
  • Ensure decorative images are marked as artifacts, not figures
  • Add or correct alt text: right-click a Figure tag > Edit Alternate Text
  • Reorder tags by dragging within the panel if reading sequence is still wrong

Section 04

Acrobat AI Assistant vs. General LLM Chat Tools.

Feature General LLM Chat Acrobat AI Assistant
How your PDF is read Raw text extracted in one pass Structured data processed with document hierarchy intact
How answers are built From whatever text was extracted From indexed chunks tied to headings, sections, and page structure
How chunks are formed Split by character or token count, regardless of meaning Split at logical document boundaries: headings, paragraphs, sections
Citations Approximate or none Links directly to location in source document
Web scraping May include web data Documents only, no web data
Hallucination risk Higher with poor structure Low, grounded in your docs only
Max documents Context window limits Up to 100 files at once
